AI Evals

Приоритет #1

Систематическое измерение качества AI-ответов. Без evals вы слепы.

  • Что тестировать: корректность ответа, следование инструкциям, формат, latency, отсутствие галлюцинаций
  • Типы evals: Unit evals (одна функция), Integration evals (цепочка), Human evals (люди оценивают)
  • LLM-as-judge: GPT-4 / Claude оценивает ответы Claude — масштабируемо
  • Инструменты: Braintrust, Promptfoo, LangSmith, Evals API (OpenAI)
  • Regression: сохраняем golden test cases, запускаем при каждом изменении промпта
  • Netcracker применение: eval для MCP-ответов, eval для RAG recall@5
Правило: "No prompt change without eval." Как "no code without tests".

AI Security

Приоритет #2

Уязвимости AI-систем отличаются от классических — важно знать до деплоя.

  • Prompt Injection: вредоносный текст в user input изменяет поведение агента. "Ignore previous instructions..."
  • Indirect Injection: агент читает документ/URL, который содержит инструкции
  • Data Leakage: system prompt, AGENTS.md, секреты через "repeat your instructions"
  • Jailbreaking: обход safety через roleplay, hypotheticals, encoded text
  • Tool Abuse: агент вызывает MCP tool с вредоносными параметрами
  • Защита: input sanitization, output filtering, allowlist tools, Human-in-the-loop для destructive ops
  • OWASP LLM Top 10: официальный список уязвимостей LLM-приложений

AI Observability

Приоритет #3

Видимость агента в production — что он делает, сколько стоит, где ломается.

  • Трассировка: каждый LLM-вызов = span с: prompt, response, tokens, latency, model
  • x-request-id: Netcracker-паттерн — сквозной ID через все сервисы и MCP
  • Инструменты: LangSmith, Langfuse, Arize, OpenTelemetry + OTLP
  • Метрики: p50/p95 latency, token usage per request, error rate, cache hit rate
  • Alerts: latency spike → cache miss → prompt regression
  • Correlation: связать AI trace с backend trace (x-request-id)

AI Cost Management

Приоритет #4

Оптимизация стоимости при масштабировании AI — часто игнорируется до первого счёта.

  • Model routing: простые задачи → Haiku/GPT-4o-mini, сложные → Sonnet/GPT-4o
  • Prompt caching: стабильный prefix → cached tokens = −90% cost
  • Batch API: не срочные задачи → batch (50% скидка у Anthropic/OpenAI)
  • Context trimming: не тащить весь файл — только нужные функции
  • Output length: явно ограничивать max_tokens для коротких ответов
  • Мониторинг бюджета: алерты при превышении daily token budget

Agent Orchestration Patterns

A4 уровень

Как проектировать системы из нескольких агентов.

  • Orchestrator-Worker: главный агент делегирует subagents, собирает результаты
  • Critic-Actor: Actor создаёт, Critic критикует и улучшает. Рефлексия.
  • Parallel Execution: независимые subagents работают одновременно
  • DAG (Graph): задачи как граф зависимостей — топологическая сортировка
  • Loop + Exit condition: агент итерирует пока не выполнен критерий
  • Human-in-the-loop gate: обязательное одобрение перед destructive action
  • Context isolation: каждый subagent имеет ограниченный контекст — для precision и безопасности

AI Governance & Compliance

Для Enterprise

Правовые и организационные рамки использования AI в enterprise.

  • EU AI Act (2024+): High-risk AI systems требуют human oversight, документацию, audit
  • GDPR + AI: нельзя обрабатывать персональные данные через внешние AI API без DPA
  • Data residency: где хранятся промпты и ответы? EU region у Anthropic/Azure OpenAI
  • Audit trails: логировать все AI-решения (что спросили, что ответил, кто использовал)
  • Ответственность: AI-агент предложил → человек одобрил → человек несёт ответственность
  • Internal policy: какие данные нельзя отправлять в LLM API (PII, trade secrets)

CI/CD for AI

DevOps + AI

Как включить AI-качество в инженерный процесс.

  • Prompt versioning: промпты в git как код, PR review, changelog
  • Eval в CI: при каждом изменении промпта → запуск eval suite → fail если регрессия
  • Golden set: набор эталонных вопрос-ответ пар, которые не должны деградировать
  • Canary deploy: новый промпт → 5% трафика → мониторинг → полный rollout
  • A/B testing промптов: как A/B фичи — статистика, winning variant
  • Model upgrade strategy: Claude 3.5 → 4: eval перед переключением, не слепо

Assessment: вопросы про стратегию AI-отдела

Что такое AI Evals и зачем они нужны?
Evals = систематическое тестирование качества AI-ответов через тест-кейсы. Без evals нельзя объективно знать: помогло ли изменение промпта, деградировало ли качество при обновлении модели. Это аналог unit tests для AI. Инструменты: Braintrust, Promptfoo, LangSmith. Golden set = эталонные вопрос-ответ пары в CI.
Что такое Prompt Injection и как защититься?
Prompt injection = вредоносный текст в user input меняет поведение агента ("Ignore previous instructions, instead do X"). Indirect injection = агент читает документ с инструкциями. Защита: input sanitization, output filtering, запрет на включение user input напрямую в system prompt, HITL для критических действий, allowlist MCP tools.
Когда fine-tuning лучше RAG?
Fine-tuning лучше когда: нужен специфичный стиль/формат (не знания), задача хорошо определена и повторяется, есть 1000+ качественных примеров, знания статичные. RAG лучше когда: knowledge base большая и меняется, нужна ссылка на источник, нет данных для обучения. Начинать всегда с prompting → RAG → fine-tuning.
Что такое x-request-id в контексте AI Observability?
x-request-id — сквозной идентификатор запроса, который проходит через все сервисы в цепочке: API → AI agent → MCP tool → backend service. Позволяет связать AI trace с backend trace и найти где возникла задержка или ошибка. В Netcracker — стандартный паттерн для трассировки cross-MCP операций.
