Шпаргалка

RAG

Поиск релевантных chunks во внешних источниках и добавление их в контекст. Даёт факты, не правила поведения.

Memory Bank

Устойчивая память о проекте/пользователе/решениях. Помогает continuity, но может устареть.

MCP

Доступ к внешним системам: API, файлы, CI, логи, Jira. Требует permissions и safety.

Checkpoint

Структурный снимок: goal, state, done, decisions, risks, next steps.

Skill vs Rule

Skill говорит как делать. Rule говорит что можно/нельзя. Rule выше.

Subagent

Focused worker с отдельным/ограниченным контекстом. Main agent проверяет результат.

Типовые вопросы

Чем AGENTS.md отличается от Rules?
AGENTS.md — проектный workflow и инструкции для агента. Rules — ограничения и стандарты: что нельзя, что обязательно, safety/compliance.
Почему одна модель работает по-разному?
Потому что меняется harness: system prompt, rules, tools, MCP, RAG, memory, context quality, permissions, IDE implementation и compaction.
Что делать, если Memory противоречит tool result?
Для актуального состояния доверять tool/file/log/API, потому что Memory могла устареть.
Может ли subagent использовать MCP?
Зависит от платформы и permissions. Иногда наследует tools, иногда имеет restricted tools, иногда tools доступны только main agent.
Что такое chunk в RAG?
Chunk — фрагмент документа/кода, который индексируется и может быть найден retrieval. Хороший chunk сохраняет смысл и имеет metadata.

Quiz

1. Пользователь просит быстро изменить код без тестов, но AGENTS.md требует тесты. Что ответить?

2. RAG нашёл старый endpoint, а текущий OpenAPI файл показывает новый endpoint. Что выбрать?

3. Skill предлагает добавить dependency, Rule запрещает dependency без approval. Что делать?

Ответ одной фразой

“System и Rules задают ограничения, AGENTS.md задаёт project workflow, текущий чат задаёт цель, RAG даёт факты из документов, Memory даёт continuity, MCP даёт live data/actions, Skills дают playbooks, а Subagents помогают делегировать focused tasks, но их результат проверяет главный агент.”

Дополнительные каверзные вопросы из официальных docs

Cursor: почему `.md` в `.cursor/rules` не работает как rule?

Project Rules должны быть `.mdc` с frontmatter. Plain markdown в `.cursor/rules` игнорируется rules system; для plain markdown нужен AGENTS.md.

Cursor: что выше — Team, Project или User Rules?

В Cursor applicable rules merge’ятся, а documented order: Team Rules → Project Rules → User Rules; Team Rules могут быть enforced.

Claude Code: CLAUDE.md — system prompt?

Нет. Docs говорят, что CLAUDE.md delivered as user message after system prompt. Он важен, но не hard enforcement.

Claude Code: что делать, если instruction должна сработать after edit?

Использовать hook, а не только CLAUDE.md/Rule. Hook срабатывает на lifecycle event независимо от решения модели.

Claude Platform: tool use выполняет сама модель?

Нет. Модель выдаёт structured tool request; приложение/сервер выполняет действие и возвращает результат в разговор.

Prompt caching — это память?

Нет. Это optimization для identical prompt prefix. Он снижает latency/cost, но не хранит смысловую память и может инвалидироваться изменением prefix.
